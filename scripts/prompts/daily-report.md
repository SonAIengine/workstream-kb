# 업무 일일 리포트 생성

{DATE} 하루 동안의 모든 업무 커뮤니케이션(Teams 채팅, Teams 채널, 메일)을 분석하여 종합 업무 리포트를 생성합니다.

**본인 이름**: {MY_NAME} (액션 아이템에서 "내가 처리할 건" vs "다른 사람에게 요청한 건" 구분 기준)

## 제외 대상
- 인사말, 단순 확인 ("네", "넵", "알겠습니다", "수고하세요")
- 빈 메시지 (content가 비어있는 것)
- 시스템 자동 메일 (noreply@, Microsoft Teams 알림, bot 발신)
- 전사 공지 메일 (HR 근태마감 등) — 행동이 필요한 경우만 포함
- 이모지/스티커만 있는 메시지
- 잡담, 점심 메뉴, 날씨 등 업무 무관 대화

## 포함 대상
- 업무 요청/지시 (누가 누구에게, 무엇을)
- 완료 보고 (배포, 계정 신청, 버그 수정 등)
- 기술 토론 (에러 분석, 아키텍처 결정, 코드 리뷰)
- 외부 커뮤니케이션 (고객사/파트너)
- 회의/일정 관련
- 공유된 기술 자료/문서 링크

## 첨부파일
- 메일 메시지에 `attachments` 배열이 있으면 해당 섹션에서 마크다운 링크로 표시
- 링크 형식: `[파일명](attachments/{DATE}/메일ID_파일명)`
- 프로젝트별 현황, 외부 대응 등 관련 섹션에 자연스럽게 포함
- 별도 "첨부파일" 섹션은 만들지 않음 — 해당 맥락에 인라인으로 삽입

## 보안
- 비밀번호, 토큰, credential, API 키 등은 반드시 `****`로 마스킹

## 응답 형식

아래 마크다운 형식으로 응답하세요. **해당 내용이 없는 섹션은 생략**합니다 (빈 섹션 출력 금지).

```markdown

date: "{DATE}"
teams_chat: N
teams_channel: N

---
# 업무 일일 리포트 - {DATE}

## 핵심 요약
- 오늘의 가장 중요한 3~5가지 사항 (bullet point)
- 하루를 한눈에 파악할 수 있는 executive summary

## 액션 아이템

### 내가 처리할 건
| 내용 | 요청자 | 출처 | 기한 |
|------|--------|------|------|

### 다른 사람에게 요청한 건
| 내용 | 담당자 | 출처 | 기한 |
|------|--------|------|------|

## 프로젝트별 현황

### {프로젝트명}
- **진행 상황**: 완료된 작업, 배포 현황
- **기술 논의**: 아키텍처 결정, 에러 대응 등
- **이슈/블로커**: 해결 필요한 문제

(프로젝트명은 chatTopic, 메일 subject, 대화 내용에서 자동 식별)

## 외부 대응

| 상대방 | 소속 | 건명 | 요약 | 대응 상태 |
|--------|------|------|------|-----------|
(고객사/파트너/외부인과의 커뮤니케이션)

## 팀 활동
- 팀원별 주요 활동 (배포, 완료 보고, 문서 작성 등)
- 채팅방에서 공유된 기술 자료/링크

## 주요 기술 메모
- 나중에 다시 찾아볼 기술 결정, 설정 정보, 에러 원인 분석
- 코드 경로, API 엔드포인트, 서버 설정 등 구체적 기술 정보

## 일정
- 확정된 회의/출장/마감
- 일정 변경 사항

## 통계

| 구분 | 채팅방 수 | 메시지 수 |
|------|----------|----------|
| Teams 채팅 | N | M |
| Teams 채널 | N | M |
| 메일 | N | M |
| **합계** | **N** | **M** |
```

## 규칙
- 원문을 그대로 복사하지 말고, 핵심만 압축
- 참여자 이름은 원문 그대로 사용
- 한국어로 작성
- front-matter의 teams_chat/teams_channel/mail 숫자는 입력 데이터의 stats에서 메시지 수를 가져올 것
- 프로젝트별 현황의 프로젝트명은 대화 내용 기반으로 자동 식별 (chatTopic, subject 등 활용)
